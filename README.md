# [보고서] 노코드 워크플로우 설계 및 비교 분석 최종 산출물

## 1. 과제 목표 및 핵심 개념 정의

- Trigger (트리거): 워크플로우를 자동으로 시작하게 만드는 '시작 이벤트'입니다. 본 기획에서는 매일 아침 구글 스프레드시트의 명부를 검사하는 스케줄러가 트리거 역할을 합니다.  

- Action (액션): 트리거가 작동한 후 시스템이 수행하는 '처리 동작'입니다. 본 과제에서는 슬랙 축하 메시지 게시와 수신자 개인 이메일(기프티콘 포함) 자동 발송의 2개 액션이 유기적으로 연결됩니다.  

- 조건 분기 (Filter / Router): 데이터의 조건에 따라 워크플로우의 경로를 나누는 필터링 역할을 합니다. 직급 필드 값을 기반으로 사원/인턴 그룹과 경력/매니저 그룹에 각기 다른 축하 문구와 카드 템플릿을 동적으로 적용합니다.  



## 2. [프로젝트 1] 자동화 도구 비교 구현 (Make vs Zapier)

동일한 기능 요구사항("구글 시트 당일 생일자 감지 -> 직급별 조건 분기 ->이메일 및 슬랙 발송")을 두 도구에 교차 구현하여 검증했습니다.  

|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
|	비교 항목								|	Make (메이크)  								|	Zapier (자피어)  
---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
|	1) UI / UX 구조							|	시각적 노드 기반 (방사형)						|	리스트 기반 (수직형)
|										| 마우스 드래그로 복잡한 멀티스텝 분기 경로를 			| 위에서 아래로 떨어지는 직렬 타임라인 리스트 형태로, 
|										| 맵 한 장에 직관적으로 시각화 가능  					| 구성 단계가 직관적임
|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
|	2) 설정 난이도							| 중~상 (학습 곡선 있음)							| 하 (매우 직관적)
|										| 모듈 간의 변수 매핑 및 인라인 함수 활용 시 				| 단계별 가이드라인이 친절하게 제공되어
|										| 초반 적응이 필요하나 자유도가 극도로 높음				| 논리적 코딩 지식이 부족해도 쉽게 세팅 가능
|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
|	3) 연동 서비스 범위						| 전 세계 수천 개의 비즈니스 소프트웨어와				| 현존하는 자동화 툴 중 가장 광범위한 서드파티
|										| 완벽히 연동되며 커넥터 세부 기능이 매우 세분화됨 			| 앱 생태계를 보유하고 있으며, 마이너한 툴 연동성이 강함
|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
|	4) 무료 플랜 범위							| 매우 합리적 (월 1,000 Ops)							| 제한적 (월 100 Tasks)
|										| 무료 플랜에서도 조건 분기(Router) 및 다단계				| 무료 플랜에서는 1:1 단단계 연동만 가능하며,
|										| 액션(Multi-step) 개수 제한 없이 무료 구성 가능			| 조건 분기(Paths) 사용 시 유료 플랜 필수
|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
|	5) 실행 로그 확인							| 실시간 시각적 버블 트래킹							| 텍스트 기반 Task History
|										| 실행 이력 히스토리에서 각 노드별 데이터 입출력 값			| 성공/실패 로그가 텍스트 리스트 형태로 차례대로
|										| (데이터 패킷)을 시각적으로 상세 디버깅 가능				| 제공되어 직관적인 데이터 역추적은 다소 투박함
|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------


###👉 [도구별 적합한 상황에 대한 의견]  

	- Make 추천 상황: 스타트업 및 중소기업처럼 예산 제약이 있으면서 복잡한 조건 분기(Router)와 
	  다단계 데이터 가공이 필요한 업무 자동화 시스템 장기 운영 시 가장 적합합니다.  

	- Zapier 추천 상황: 초기 러닝 커브 없이 즉시 실무에 적용해야 하는 경우, 혹은 유료 구독 예산이 
	  확보되어 정형화된 비즈니스 툴을 빠르게 결합하고자 하는 대기업 업무 부서에 적합합니다.  



## 3. [프로젝트 2] 자유 주제 자동화 설계 및 구현

	### - 반복 업무 정의: 매달 임직원 생일을 수동 확인하고, 당일 아침마다 축하 메시지를 디자인하여 사내 채널 및 메일로 노가다 발송하던 반복 업무.  
	
	### - 선정 도구: Make (메이크)  
		- 이유: 무료 플랜에서도 무제한 다단계 액션 및 시각적 라우터(Router) 조건 분기를 완벽히 지원하여 과금 리스크가 전혀 없기 때문입니다.  

	### - 워크플로우 흐름 및 스프레드시트 수식 전처리:  
		- 구글 스프레드시트에 오늘 날짜와 생일(MM-DD)이 일치할 경우 Y를 반환하는 전처리 수식을 삽입하여 데이터 효율을 극대화합니다.

  		=IF(TEXT(TODAY(),"MM-DD")=TEXT(D2,"MM-DD"), "Y", "N")}


		- Step 1 (Trigger): Google Sheets - Search Rows 모듈이 매일 아침 09:00에 실행되어 '오늘여부' 컬럼 값이 Y인 생일 직원의 데이터만 정확히 스캔합니다.  
		- Step 2 (Router): 가져온 데이터를 중앙 라우터에 통과시켜 '직급' 필드 값에 따라 두 갈래의 독립 경로로 분기합니다.  
		- Step 3 (경로 A - 인턴/신입 그룹): 직급이 인턴·신입인 경우, 격려와 환영을 담은 산뜻한 텍스트 템플릿을 생성하여 
		           슬랙(Create a Message) 공용 채널에 게시하고 지메일(Send an email)을 통해 스타벅스 기프티콘과 카드를 자동 발송합니다.  
		- Step 4 (경로 B - 경력/매니저 그룹): 직급이 경력·매니저인 경우, 노고에 대한 감사와 존경을 담은 중후한 텍스트 템플릿으로 
			   슬랙 채널에 전사 공지하고 지메일로 축하 메시지를 자동 발송합니다. 



## 4. 보너스 과제 적용 명세 (선택 사양 탑재)  

	### -  보너스 1 (AI 연동 Action): Make의 라우터 직전에 ChatGPT(OpenAI) 모듈을 배치하여, 구글 시트에서 가져온 직원의 이름과 직급 데이터를 프롬프트에 동적으로 바인딩합니다. 
                       		이를 통해 매년 똑같은 문구가 아닌, AI가 매번 새롭게 생성한 개인 맞춤형 문장(예: "홍길동 사원님, 지난 프로젝트에서의 기여에 감사하며...")을 자동 생성해 이메일에 
                      		이식하도록 설계 고도화를 달성했습니다.  

	### - 보너스 2 (오류 알림 및 실패 방지): 만약 아침 9시 실행 중 Gmail API 일일 제한 초과나 기프티콘 번호 누락 오류가 발생할 경우, 워크플로우가 중단되지 않고 곧바로 대체 경로인 
                       		Slack 관리자 단독 알림 채널로 즉각적인 에러 로그 메시지가 발송되도록 트래킹 구조를 이식했습니다.
